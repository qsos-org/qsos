## Purpose

This phase consists of weighting criteria according to your priorities. You define the importance of each grid element through requirement presets, which will be applied during comparison.

In this phase, you create **requirement presets**: sets of weights for grid sections, subsections, and criteria.

## Requirement Presets

A requirement preset assigns a weight to each element of an evaluation grid. This weight determines the element's influence on the final score.

You can create multiple presets for the same grid according to your usage contexts.

## Weighting

Each grid element is associated with a numeric weight: sections, subsections, and criteria. The higher the weight, the more the element influences the final score.

By default, all elements have a weight of 100, which amounts to a simple average.

**Formula**: `Final Score = (∑ score × weight) / (∑ weights)`

Example:

| Criterion | Score | Weight | Score × Weight |
|-----------|-------|--------|----------------|
| Heritage | 2 | 200 | 400 |
| Activity | 2 | 250 | 500 |
| Governance | 1 | 150 | 150 |
| Industrialization | 2 | 100 | 200 |
| **Total** | | **700** | **1250** |

Final Score = 1250 / 700 = **1.79**
