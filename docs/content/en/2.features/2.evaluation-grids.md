**Evaluation grids** are the **central component** of the QSOS methodology. They provide a **structured** way to evaluate open source software according to specific and reproducible criteria. This is **Phase 2** of the methodology.

---

## STEP 1: Create a Grid

Create a grid from a software type. The **QSOS 2.0 Maturity section** is added automatically (immutable and locked).

::doc-image
---
src: /images/evaluation-grids/create-evaluation-grid-en.png
alt: Creating an evaluation grid
caption: Figure 1 — Evaluation grid creation form
maxWidth: 800px
---
::

Fill in the grid **name** and **description**.

<details>
<summary>View JSON</summary>

```json
{
  "gridVersion": "1.0",
  "qsosVersion": "2.0",
  "creatorEmail": "alice@company.fr",
  "createdAt": "2025-09-08T14:30:00.578Z",
  "updatedAt": "2025-09-08T14:30:00.578Z",
  "changeLog": "Initial version",
  "sections": [
    {
      "ref": "maturity",
      "name": "qsos_maturity_2_0.maturity",
      "description": "QSOS Maturity Section",
      "sections": []
    }
  ]
}
```

</details>

---

## STEP 2: Add Sections and Criteria

Add your custom sections (Performance, Maintainability, Security, etc.) with criteria scored on a 0-1-2 scale. You can create sub-sections for better hierarchy.

::doc-image
---
src: /images/evaluation-grids/add-criteria-en.png
alt: Adding criteria
caption: Figure 2 — Adding sections and criteria
maxWidth: 800px
---
::

Validation: minimum 3 sections (Maturity included), each criterion must have a description for scores 0, 1 and 2.

<details>
<summary>View JSON</summary>

```json
{
  "gridVersion": "1.0",
  "qsosVersion": "2.0",
  "creatorEmail": "alice@company.fr",
  "createdAt": "2025-09-08T14:30:00.578Z",
  "updatedAt": "2025-09-08T14:30:00.578Z",
  "changeLog": "Initial version",
  "sections": [
    {
      "ref": "maturity",
      "name": "qsos_maturity_2_0.maturity",
      "description": "QSOS Maturity Section",
      "sections": []
    },
    {
      "ref": "perf-section",
      "name": "Performance",
      "description": "Performance metrics",
      "sections": [
        {
          "ref": "client-reactivity",
          "name": "Client-side reactivity",
          "description": "Responsiveness to client changes",
          "desc0": "Refresh time > 500ms",
          "desc1": "Refresh time 100-500ms",
          "desc2": "Refresh time < 100ms"
        }
      ]
    },
    {
      "ref": "maint-section",
      "name": "Maintainability",
      "description": "Code quality and maintainability",
      "sections": [
        {
          "ref": "code-doc",
          "name": "Code documentation",
          "description": "Quality of source code documentation",
          "desc0": "No documentation",
          "desc1": "Basic documentation",
          "desc2": "Excellent documentation"
        }
      ]
    }
  ]
}
```

</details>

## STEP 3: Save and Version

The saved grid is **immutable** (v1.0). To modify it, create a new version (patch, minor or major) with a changelog.

::doc-image
---
src: /images/evaluation-grids/edit-evaluation-grid-en.png
alt: Editing an evaluation grid
caption: Figure 3 — Grid versioning form
maxWidth: 800px
---
::

<details>
<summary>View JSON - v1.1</summary>

```json
{
  "gridVersion": "1.1",
  "qsosVersion": "2.0",
  "changelog": "Clarified desc2 for Client-side reactivity criterion"
}
```

</details>

---

## STEP 4: Use a Grid to Evaluate

Select a grid from a software version to create an evaluation. For each criterion, assign a score (0, 1 or 2) with a justification and sources.

::doc-image
---
src: /images/evaluation-grids/add-evaluation-en.png
alt: Adding an evaluation
caption: Figure 4 — Evaluating with a grid
maxWidth: 800px
---
::

<details>
<summary>View JSON</summary>

```json
{
  "evaluationUid": "eval-react-19-perf",
  "authors": [
    {
      "name": "Alice Dupont",
      "email": "alice@company.fr"
    }
  ],
  "language": "en",
  "createdAt": "2025-09-08T15:30:00.578Z",
  "updatedAt": "2025-09-08T15:30:00.578Z",
  "sections": [
    {
      "sections": {
        "0": {
          "sections": {
            "0": {
              "score": 2,
              "comment": "React 19 with concurrent rendering. Sub-millisecond updates."
            },
            "1": {
              "score": 2,
              "comment": "FCP < 250ms on homepage at 5G"
            }
          }
        },
        "1": {
          "sections": {
            "0": {
              "score": 2,
              "comment": "Less than 10 dependencies"
            }
          }
        }
      }
    }
  ]
}
```

</details>

## Best Practices

- Keep criteria measurable and objective
- Document scoring rationale
- Review and update grids regularly
- Involve multiple evaluators for objectivity
