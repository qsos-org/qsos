---
title: Requirements Presets
description: Create and manage weighting presets for QSOS comparisons
---

**Requirements presets** allow you to save and reuse **custom weightings** when comparing software. This is **Phase 3** of the methodology.

A preset defines the relative importance of each evaluation criterion according to your organizational priorities (e.g., security matters more than performance in a banking context).

---

## STEP 1: Create a preset

Create a preset from the **presets view** or directly from a **comparison page**.

::doc-image
---
src: /images/presets/create-preset-en.png
alt: Creating a preset
caption: Figure 1 — Creating a preset
maxWidth: 350px
---
::

Fill in a clear **name** and optionally a **description** to specify the usage context (e.g., "Security priority - banking sector"). The preset is linked to a specific **grid version** and **software type**.

<details>
<summary>View JSON</summary>

```json
{
  "presetUid": "preset-001",
  "label": "Security priority",
  "description": "Security-oriented weighting for the banking sector",
  "userUid": "alice@company.fr",
  "gridVersion": "1.0",
  "softwareTypeUid": "web_frameworks",
  "criteriaWeights": {
    "sections": [
      {
        "weight": 800,
        "sections": [
          { "weight": 700 },
          { "weight": 900 }
        ]
      },
      {
        "weight": 500,
        "sections": [
          { "weight": 400 }
        ]
      }
    ]
  },
  "createdAt": "2025-09-08T14:00:00.000Z",
  "updatedAt": "2025-09-08T14:00:00.000Z"
}
```

</details>

## Best Practices

- Name your presets explicitly (context + priority)
- Create different presets for different business contexts
- Reuse presets for consistent comparisons over time
- Update your presets when the grid evolves
