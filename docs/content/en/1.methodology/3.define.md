## Purpose

To evaluate effectively, you must first define what to evaluate. You organize your software into categories (types) and identify the versions to evaluate.

In this phase, you create:

- **Software types**: functional categories (web frameworks, databases, etc.)
- **Software**: the software to evaluate with their SPDX license
- **Versions**: the specific versions of each software

## Software Types

A software type corresponds to a functional domain. It groups software that addresses the same need.

Examples:

- **BI** (Business Intelligence): dashboards, visualization, reporting
- **Web Frameworks**: frontend and backend development
- **ETL**: data integration and transformation
- **Databases**: relational or NoSQL storage
- **CMS**: content management and collaboration

## Software

Within each type, you add the relevant software.

Each software is associated with:

- **An SPDX license**: the international standard that references free and open source licenses
- **One or more versions**: the versions you wish to evaluate

## Next Step

Once your types and software are defined, move to **[Phase 2: Evaluate](/en/methodology/evaluate)** to define criteria and begin scoring.
