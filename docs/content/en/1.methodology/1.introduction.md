# QSOS: Qualification and Selection of Open Source Software

## Why QSOS?

Selecting open source software is complicated. You need to assess code quality, project viability, governance, security... And you must justify your choice to your teams and management.

QSOS provides a simple and transparent method to evaluate and compare open source projects. Instead of choosing based on popularity or intuition, you evaluate against objective criteria, document your scores with evidence, and generate clear reports.

Read the [QSOS Manifesto](/en/methodology/qsos-manifesto) to understand the foundations of the methodology.

## Who Is QSOS For?

- People curious to learn about the method for professional or personal purposes
- Open source project communities
- IT experts wishing to learn and apply the method in their daily work of evaluating and selecting components to build software solutions that meet their needs or those of their clients

## How Does QSOS Work

QSOS operates through four distinct phases, each building on the previous one:

### Phase 1: Define
Establish the scope of your evaluation by creating software types (functional categories). For each type, add the software to evaluate with their specific versions and corresponding SPDX licenses.

### Phase 2: Evaluate
Create an evaluation grid for each software type by defining criteria, then evaluate each software according to the grid. Each criterion receives a score (0 = absent, 1 = partial, 2 = complete) with justification.

### Phase 3: Qualify
Create requirements presets by defining the weights of sections, subsections and criteria of a grid. These weights will be used to weight evaluations during comparison.

### Phase 4: Select
Compare results by selecting software and applying your requirements presets. Visualize differences with charts and export results as PDF.
