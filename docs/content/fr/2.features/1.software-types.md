---
title: 'Types de logiciels'
description: 'Créer et gérer les catégories de logiciels dans QSOS'
---

QSOS organise les logiciels en **catégories** pour fournir des critères d'évaluation plus pertinents. Chaque type de logiciel dispose de grilles d'évaluation adaptées à ses caractéristiques. C'est la **Phase 1** de la méthodologie.

---

## ÉTAPE 1: Créer un Type

Un **type de logiciel** regroupe des solutions comparables répondant à un même besoin, comme *Frameworks web*, *Bases de données* ou *Outils de monitoring*.

::doc-image
---
src: /images/software-types/create-software-type-fr.png
alt: Création d'un type de logiciel
caption: Figure 1 — Formulaire de création d'un type de logiciel
maxWidth: 350px
---
::

Remplissez un **nom** clair et une **description** qui définit le périmètre du type. Vous pouvez aussi ajouter une **icône** pour l'identification visuelle dans les listes.

<details>
<summary>Voir le JSON</summary>

```json
{
  "uid": "web_frameworks",
  "name": "Web frameworks",
  "description": "Frameworks de développement pour applications web",
  "icon": "",
  "creatorEmail": "alice@company.fr",
  "createdAt": "2025-09-08T13:44:00.578Z",
  "updatedAt": "2025-09-08T13:44:00.578Z"
}
```

</details>

---

## ÉTAPE 2: Ajouter un Logiciel

Une fois le type créé, ajoutez les logiciels que vous souhaitez évaluer et comparer.

::doc-image
---
src: /images/software-types/create-software-fr.png
alt: Création d'un logiciel
caption: Figure 2 — Formulaire de création d'un logiciel
maxWidth: 350px
---
::

Pour chaque logiciel, remplissez son **nom**, une courte **description**, sa **licence** au format SPDX (ex: MIT, Apache-2.0), son **URL** officielle, et optionnellement une **URL de démo** et une **icône**.

<details>
<summary>Voir le JSON</summary>

```json
{
  "uid": "react",
  "name": "React",
  "description": "Bibliothèque pour interfaces utilisateur web et natives",
  "licenseId": "MIT",
  "url": "https://react.dev/",
  "demoUrl": "https://react.dev/learn",
  "icon": "icon.png"
}
```

</details>

---

## ÉTAPE 3: Ajouter une Version

Chaque logiciel peut avoir plusieurs **versions** correspondant à différents états du projet. Les évaluations se font sur une version spécifique.

::doc-image
---
src: /images/software-types/add-software-version-fr.png
alt: Ajout d'une version
caption: Figure 3 — Formulaire d'ajout d'une version
maxWidth: 350px
---
::

Remplissez le **numéro de version** (ex: `19`, `3.2.1`) et un **résumé** des changements majeurs.

<details>
<summary>Voir le JSON</summary>

```json
{
  "version": "19",
  "summary": "Introduction des Actions et des Server Components",
  "dateAdded": "2025-09-08T13:44:00.578Z"
}
```

</details>

---

## Bonnes pratiques

- Noms clairs et descriptifs pour les types
- Licences SPDX officielles uniquement
- Une version = un état du projet (ne pas mélanger branches/forks)
