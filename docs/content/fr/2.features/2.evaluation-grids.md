---
title: Grilles d'évaluation
description: Créer et utiliser les grilles d'évaluation dans QSOS
---

Les **grilles d'évaluation** sont le **composant central** de la méthodologie QSOS. Elles fournissent une manière **structurée** d'évaluer les logiciels open source selon des critères spécifiques et reproductibles. C'est la **Phase 2** de la méthodologie.

---

## ÉTAPE 1: Créer une Grille

Créez une grille depuis un type de logiciel. La **section Maturité QSOS 2.0** s'ajoute automatiquement (immuable et verrouillée).

::doc-image
---
src: /images/evaluation-grids/create-evaluation-grid-fr.png
alt: Création d'une grille d'évaluation
caption: Figure 1 — Formulaire de création d'une grille d'évaluation
maxWidth: 800px
---
::

Remplissez le **nom** et la **description** de la grille.

<details>
<summary>Voir le JSON</summary>

```json
{
  "gridVersion": "1.0",
  "qsosVersion": "2.0",
  "creatorEmail": "alice@company.fr",
  "createdAt": "2025-09-08T14:30:00.578Z",
  "updatedAt": "2025-09-08T14:30:00.578Z",
  "changeLog": "Version initiale",
  "sections": [
    {
      "ref": "maturity",
      "name": "qsos_maturity_2_0.maturity",
      "description": "Section Maturité QSOS",
      "sections": []
    }
  ]
}
```

</details>

---

## ÉTAPE 2: Ajouter des Sections et Critères

Ajoutez vos sections personnalisées (Performance, Maintenabilité, Sécurité, etc.) avec des critères notés sur une échelle 0-1-2. Vous pouvez créer des sous-sections pour une meilleure hiérarchie.

::doc-image
---
src: /images/evaluation-grids/add-criteria-fr.png
alt: Ajout de critères
caption: Figure 2 — Ajout de sections et critères
maxWidth: 800px
---
::

Validation : minimum 3 sections (Maturité comprise), chaque critère doit avoir une description pour les scores 0, 1 et 2.

<details>
<summary>Voir le JSON</summary>

```json
{
  "gridVersion": "1.0",
  "qsosVersion": "2.0",
  "creatorEmail": "alice@company.fr",
  "createdAt": "2025-09-08T14:30:00.578Z",
  "updatedAt": "2025-09-08T14:30:00.578Z",
  "changeLog": "Version initiale",
  "sections": [
    {
      "ref": "maturity",
      "name": "qsos_maturity_2_0.maturity",
      "description": "Section Maturité QSOS",
      "sections": []
    },
    {
      "ref": "perf-section",
      "name": "Performance",
      "description": "Métriques de performance",
      "sections": [
        {
          "ref": "client-reactivity",
          "name": "Réactivité côté client",
          "description": "Réactivité aux changements côté client",
          "desc0": "Temps de rafraîchissement > 500ms",
          "desc1": "Temps de rafraîchissement 100-500ms",
          "desc2": "Temps de rafraîchissement < 100ms"
        }
      ]
    },
    {
      "ref": "maint-section",
      "name": "Maintenabilité",
      "description": "Qualité et maintenabilité du code",
      "sections": [
        {
          "ref": "code-doc",
          "name": "Documentation du code",
          "description": "Qualité de la documentation du code source",
          "desc0": "Pas de documentation",
          "desc1": "Documentation basique",
          "desc2": "Documentation excellente"
        }
      ]
    }
  ]
}
```

</details>

## ÉTAPE 3: Sauvegarder et Versionner

La grille sauvegardée est **immuable** (v1.0). Pour la modifier, créez une nouvelle version (patch, minor ou major) avec un changelog.

::doc-image
---
src: /images/evaluation-grids/edit-evaluation-grid-fr.png
alt: Édition d'une grille d'évaluation
caption: Figure 3 — Formulaire de versioning de la grille
maxWidth: 800px
---
::

<details>
<summary>Voir le JSON - v1.1</summary>

```json
{
  "gridVersion": "1.1",
  "qsosVersion": "2.0",
  "changelog": "Clarification de la desc2 pour le critère Réactivité côté client"
}
```

</details>

---

## ÉTAPE 4: Utiliser une Grille pour Évaluer

Sélectionnez une grille depuis une version de logiciel pour créer une évaluation. Pour chaque critère, attribuez un score (0, 1 ou 2) avec une justification et des sources.

::doc-image
---
src: /images/evaluation-grids/add-evaluation-fr.png
alt: Ajout d'une évaluation
caption: Figure 4 — Évaluation avec une grille
maxWidth: 800px
---
::

<details>
<summary>Voir le JSON</summary>

```json
{
  "evaluationUid": "eval-react-19-perf",
  "authors": [
    {
      "name": "Alice Dupont",
      "email": "alice@company.fr"
    }
  ],
  "language": "fr",
  "createdAt": "2025-09-08T15:30:00.578Z",
  "updatedAt": "2025-09-08T15:30:00.578Z",
  "sections": [
    {
      "sections": {
        "0": {
          "sections": {
            "0": {
              "score": 2,
              "comment": "React 19 avec concurrent rendering. Mises à jour sub-milliseconde."
            },
            "1": {
              "score": 2,
              "comment": "FCP < 250ms sur la page d'accueil à 5G"
            }
          }
        },
        "1": {
          "sections": {
            "0": {
              "score": 2,
              "comment": "Moins de 10 dépendances"
            }
          }
        }
      }
    }
  ]
}
```

</details>

## Bonnes pratiques

- Garder les critères mesurables et objectifs
- Documenter la justification des notes
- Réviser et mettre à jour les grilles régulièrement
- Impliquer plusieurs évaluateurs pour l'objectivité
